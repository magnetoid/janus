# Journal 2026-06-08



## 02:13 · decision

Rebrand strategy for Hermes→Janus: 3 distinct layers must be handled differently. (1) DISPLAY (safe, done in Stage 1): skin_engine branding, banner wordmark, prompt_builder DEFAULT_AGENT_IDENTITY, display-name fallbacks. (2) PROTOCOL IDENTITY (outbound attribution, test-coupled): OpenRouter X-Title in agent/auxiliary_client.py, MCP OAuth client_name in tools/mcp_oauth.py, ACP registry name, codex client_title — changing these alters how the app identifies to external services. (3) RUNTIME IDENTITY (breaks installs without shims): HERMES_HOME env var + get_hermes_home() in hermes_constants.py, package name hermes-agent in pyproject.toml, python modules hermes_cli/hermes_constants/hermes_state, ~/.hermes config dir. A blind sed is unsafe — esp. since modules need file renames + import updates and ~/.hermes holds all user data. Banner emblem (JANUS_EMBLEM) is still the Hermes caduceus — needs bespoke art.

## 02:30 · decision

User decision (2026-06-08): rebrand goes ALL-IN, no back-compat shims — this is a fresh fork with no existing users. Rename hermes→janus everywhere including python modules (hermes_cli→janus_cli, hermes_constants→janus_constants, hermes_state, hermes_logging, hermes_time, hermes_bootstrap), env vars (HERMES_HOME→JANUS_HOME), functions (get_hermes_home→get_janus_home), config dir (~/.hermes→~/.janus), package (hermes-agent→janus), CLI entries (hermes→janus). Case-preserving: HERMES→JANUS, Hermes→Janus, hermes→janus. Exclude .git/.torsor/node_modules/venvs and lockfiles (uv.lock/package-lock.json need regen via `uv lock`/`npm install`, not sed). First best-of-both feature chosen: daily memory snapshots (OpenClaw-style dated workspace/memory/*.md layer over Hermes memory).

## 13:32 · learning

Full Janus test suite result (29,431 tests): 29,084 passed, 81 failed. Triage: only TWO clusters (13 tests) were real rebrand regressions, both fixed in 06a6431:
(1) janus_cli/model_switch.py — the hermes->janus sweep renamed the Nous Hermes model-detection REGEX (`hermes[-_ ]?[34]` became `janus...`), so is_nous_janus_non_agentic() stopped matching real Nous Research Hermes 3/4 chat models. Lesson: the protect-list guarded model-ID string LITERALS but missed a bare 'hermes' inside a REGEX (followed by '[', not a digit). When rebranding, also grep for 'hermes' in matching/detection logic, not just data strings.
(2) janus_cli/banner.py — Stage 1 hand-edit set 'Janus v' but the sweep made everything else 'Janus Agent v'; aligned to 'Janus Agent v'.
All other 68 failures are ENVIRONMENTAL, not rebrand: missing optional deps (agent_client_protocol/modal/voice/wecom-defusedxml), macOS-specifics (/tmp->/private/tmp symlink, systemd/service mgmt), real Claude Code OAuth creds leaking from the host machine into credential-resolution tests, and absent .github/workflows in this checkout. These fail on the original Hermes baseline too on this machine.
